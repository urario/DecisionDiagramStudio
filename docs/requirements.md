## 1. プロジェクト概要

### 1.1 目的

Decision Diagram Studio（以下、本アプリ）は、BDD・ZDD・MTBDD・ZMTBDD を対話的に学習・実験・可視化できる Windows デスクトップアプリケーションである。

コアロジックは既存ライブラリ `DecisionDiagramSharp`（以下、ライブラリ）に委譲し、本アプリは UI・UX・ビジュアライゼーション・エクスポートに専念する。

### 1.2 対象ユーザー

| ユーザー種別 | 目的 |
|---|---|
| 学習者 | BDD/ZDD の基礎概念をグラフィカルに理解したい |
| 研究者・実験者 | 式を変えながら図の変化を即座に確認したい |
| 教育者 | 授業・資料用に図や真理値表をエクスポートしたい |
| OSS 開発者 | DecisionDiagramSharp の動作を GUI で確認したい |
| レガシーコード担当者 | インクルード依存グラフをビジュアルで把握したい |

### 1.3 スコープ境界

**本アプリに含むもの:**

- WinUI 3 デスクトップ GUI
- 4 種類の図ファミリー（BDD / ZDD / MTBDD / ZMTBDD）の対話的操作
- グラフビジュアライゼーション（DOT → SVG レンダリング）
- 真理値表・集合族・値テーブルの表示
- 削減前後（BDT vs BDD）の比較表示
- CSV / Markdown / AsciiDoc エクスポート（TT コピー）
- SVG エクスポート
- プリセット式の提供（学習支援）
- 解説パネル（右サイドペイン）

**本アプリに含まないもの:**

- 新しい Decision Diagram アルゴリズムの実装（ライブラリの責務）
- Graphviz プロセスの管理はアプリ層で担うが、Core には入れない
- クラウド連携・マルチユーザー機能
- MDD / ADD（ライブラリロードマップ外）

### 3.2 タイトルバー・ツールバー

#### タイトルバー

- アプリ名「Decision Diagram Studio」を常時表示する。
- 現在開いている図のコンテキスト（例: `BDD - 3変数`）をサブタイトルとして表示する。
- 現在の関数シグネチャ（例: `f f : {0,1}^3 → {0,1} on-set 5`）を右端に表示する。

#### ツールバー（左グループ）

| ボタン | アイコン | 動作 |
|---|---|---|
| 新規 | ＋ | ワークベンチをリセットし空の状態で開始する |
| ランダム | サイコロ | ランダムなブール関数を生成して表示する |

#### ツールバー（中央グループ）

| ボタン | アイコン | キーボード | 動作 |
|---|---|---|---|
| 元に戻す | ← | Ctrl+Z | 直前の操作を取り消す（Undo） |
| やり直し | → | Ctrl+Y | 取り消しを元に戻す（Redo） |

#### ツールバー（右グループ）

| ボタン | アイコン | 動作 |
|---|---|---|
| TT コピー | 表コピー | 真理値表をクリップボードにコピーする（形式選択可） |
| SVG | ↓ | 現在のグラフを SVG ファイルとして保存する |

### 3.3 左ナビゲーションペイン

WinUI 3 の `NavigationView`（左サイドバー）を使用する。

| 項目 | バッジ | 遷移先 |
|---|---|---|
| ワークベンチ | なし（選択中を強調） | WorkbenchPage |
| 真理値表 | on-set 件数（例: 5） | TruthTablePage |
| 変数順序 | なし | VariableOrderPage |
| 統計 | 削減ノード数等（例: 4） | StatisticsPage |
| その他 | なし | 将来拡張用 |
| 設定 | なし | SettingsPage |

> **注:** バッジ値の正確な意味（on-set 件数・削減ノード数等）は実装時に確定する（OQ-005）。

### 3.4 左サイドパネル（WorkbenchPage の入力エリア）

#### 3.4.1 関数定義表示

- 現在の関数定義を読み取り専用テキストで表示する（例: `f f : {0,1}^3 → {0,1} on-set 5`）。
- グラフビューのヘッダーおよびタイトルバー右端にも同じシグネチャを表示する。

#### 3.4.2 種別（図ファミリー選択）

- 4 択のラジオボタングループで表示する:
  - `● BDD`（青色インジケータ）
  - `○ ZDD`（緑色インジケータ）
  - `○ MTBDD`（橙色インジケータ）
  - `○ ZMTBDD`（紫色インジケータ）
- 選択変更時は即座に図ファミリーを切り替える（確認ダイアログなし）。
- インジケータの色はグラフビューヘッダーの色と一致させる。

#### 3.4.3 真理値表（インライン表示）

左サイドパネル内に真理値表をコンパクトに表示する。

- 列: `#`（行番号）・変数名列（例: `a`・`b`・`c`）・`f`（関数値）。
- 行は 0 から始まる 2 進インデックス順（LSB-first）で表示する。
- `f = 1` の行のセルは青色背景で強調表示する（on-set の可視化）。
- フッター行に `on-set: N/2^M　M 変数 · 2^M=N` のサマリーを表示する。
- 変数数が多い場合（> 4 変数）はスクロールを有効にする。
- 各行の `f` 列セルをクリックして 0/1 をトグルすることで真理値表を編集できる。

#### 3.4.4 プリセット

よく使われる式を即座に呼び出せるプリセットボタン群を表示する。

モックで確認されたプリセット例:

- `f = ab + c`
- `Majority(a,b,c)`
- `a = b = c`
- `a · b · c`

プリセットをクリックすると式・変数・真理値表が即座に更新される。

#### 3.4.5 変数順序

変数の順序を上下矢印ボタンで変更できるリストを表示する。

- 各行の形式: `[順序番号] [変数名] [↑] [↓]`
- 変数順序の変更はリアルタイムで図に反映する。

### 3.5 グラフビュー（中央ペイン）

#### 3.5.1 ヘッダー

- 図ファミリーの色付きインジケータと正式名称を表示する（例: `● BDD  Binary Decision Diagram`）。
- 右上に切り替えボタン `[削減前(BDT)]` `[削減後]` を表示する。

#### 3.5.2 削減前後の切り替え（本アプリの核心学習機能）

| ボタン | 表示内容 |
|---|---|
| 削減前（BDT） | 削減ルール適用前の完全展開ツリー（Binary Decision Tree）を表示する |
| 削減後 | 削減ルール適用後の正規 BDD を表示する |

- 両ボタンを切り替えることで削減の効果を直感的に学習できる。
- 削減前後の差分はステータスバーの統計値（ノード数・冗長数・共有数）で数値的に確認できる。
- 初期状態は「削減後」を選択状態にする。

#### 3.5.3 グラフ描画仕様

| 要素 | 形状・スタイル |
|---|---|
| 内部ノード（変数ノード） | 楕円形・変数名テキスト |
| 終端ノード 0 | 矩形・白背景・「0」テキスト |
| 終端ノード 1 | 矩形・青背景・「1」テキスト |
| Low エッジ（0分岐） | 破線 |
| High エッジ（1分岐） | 実線 |

- ノードをクリックすると右解説パネルに当該ノードの説明が表示される。
- ノードにカーソルを合わせるとツールチップで変数名・分岐条件を表示する。

#### 3.5.4 グラフ操作

- マウスホイールによるズームイン・アウト。
- ドラッグによるパン（画面移動）。
- ダブルクリックでビューをリセット（全体表示）。

### 3.6 右解説パネル

学習支援のためのコンテキスト説明エリア。

#### 3.6.1 通常状態

- 上部に「解説」の見出しを表示する。
- ノード未選択時: 「図のノードをクリックすると、ここに意味と分岐の説明が表示されます。」というプレースホルダーを表示する。
- ノード選択時: 選択ノードの変数名・分岐条件（Low/High の意味）・到達条件を表示する。

#### 3.6.2 図ファミリーの説明テキスト

選択中の図ファミリーの基本説明を常時表示する。

BDD の例:
> **BDD とは**  
> 関数 f を、変数の値に応じた分岐の連鎖として表現。冗長ノード削除と同型ノード共有で圧縮されます。

各図ファミリーに対して同様の説明テキストを用意する。

#### 3.6.3 Tweaks パネル

- 右解説パネルの下部に折りたたみ可能な「Tweaks」パネルを設ける（`[Tweaks] [×]` ヘッダー付き）。
- 開発者・パワーユーザー向けの調整項目を提供する。
- 初版での提供項目: テーマ切り替え（ライト / ダーク）。
- 製品版では設定ページに統合するか隠し機能として残すか検討する（OQ-011）。

### 3.7 ステータスバー

画面最下部に統計情報を横並びで表示する。

| 項目 | 表示例 | 説明 |
|---|---|---|
| ノード数 | `5　↓4削減` | 削減後のノード数と削減されたノード数 |
| 冗長 | `3　↓4削減` | 冗長（Low==High）ノードの数と削減数 |
| 終端 | `2` | 終端ノードの数（0 と 1 の合計） |
| 共有 | `2` | 共有（同型）ノードの数 |

- 削減前後の切り替え時にリアルタイムで更新する。
- 削減数は矢印付きで表示し、削減効果を可視化する。

## 4. 機能要件

### 4.1 図ファミリー管理

#### FR-001 ファミリー選択

- ユーザーは BDD / ZDD / MTBDD / ZMTBDD のいずれかをラジオボタンで選択できる。
- 選択に応じてグラフビュー・真理値表・解説テキストが切り替わる。

#### FR-002 変数管理

- 変数名をテキストで定義できる（例: `a`・`b`・`c`）。
- 変数順序を上下矢印ボタンで変更できる。
- 変数順序の変更はリアルタイムで図に反映する。
- 変数名は `DecisionDiagramManager` の `GetOrAddVariable(name)` を介して登録する。

### 4.2 BDD 操作

#### FR-BDD-001 真理値表による関数定義

- 真理値表を左サイドパネルのインライン表で編集できる（`f` 列セルのクリックで 0/1 をトグル）。
- 入力した真理値表から BDD を構築する。
- 関数シグネチャ（`f : {0,1}^n → {0,1} on-set N`）を自動的に生成する。

#### FR-BDD-002 プリセット呼び出し

- 定義済みプリセットを選択すると真理値表と変数が即座に設定される。
- `PresetService` でプリセットを管理し、将来の拡張が容易な設計にする。

#### FR-BDD-003 ランダム生成

- ツールバーの「ランダム」ボタンを押すと現在の変数数でランダムなブール関数を生成する。
- 生成後は Undo で元に戻せる。

#### FR-BDD-004 削減前後の表示

- 「削減前（BDT）」: 削減ルールを適用しない完全展開ツリーを表示する。
- 「削減後」: 通常の削減済み BDD を表示する。
- 両状態のノード数差分をステータスバーで表示する。

> **実装注意:** ライブラリは削減前ツリーの生成 API を持たないため、`DiagramService` でアプリ層として BDT を生成する（OQ-001）。

#### FR-BDD-005 統計情報の表示

- ノード数・冗長ノード数（Low==High）・終端数・共有ノード数をステータスバーに常時表示する。
- `ToStatisticsTable()` を使用する。

### 4.3 ZDD 操作

#### FR-ZDD-001 集合族入力

- 集合を文字列で入力できる。
- `ZddManager.MakeFamily(IEnumerable<IEnumerable<string>>)` を使用する。

#### FR-ZDD-002 集合族操作

- Union / Intersect / Difference / Subset0 / Subset1 / Containing / NotContaining を操作できる。
- 操作結果を新しいダイアグラムとして表示する。

#### FR-ZDD-003 集合族・統計表示

- 集合族を表形式で表示し `CountSets` の結果を表示する。

### 4.4 MTBDD 操作

#### FR-MTBDD-001 整数値真理値表入力

- 整数値の真理値表を入力できる（LSB-first 変数順序）。
- `MtbddManager.Create(int[])` を使用する。

#### FR-MTBDD-002 評価・表示

- 変数への真偽値を入力し整数結果を即時表示できる。
- 値テーブルを表形式で表示する。

### 4.5 ZMTBDD 操作

#### FR-ZMTBDD-001 スパース整数値入力

- MTBDD と同様に整数値テーブルを入力できる。
- `ZmtbddManager.Create(int[])` を使用する。

#### FR-ZMTBDD-002 ゼロ抑圧の可視化

- MTBDD と ZMTBDD のノード数比較を表示しゼロ抑圧の効果を可視化する。

### 4.6 グラフビジュアライゼーション

#### FR-VIZ-001 DOT → SVG レンダリング

- `ToDot()` で取得した DOT テキストを Graphviz `dot` コマンドで SVG に変換し WebView2 で表示する。
- Graphviz が見つからない場合は DOT テキストをフォールバック表示する。

#### FR-VIZ-002 ノードクリック・解説連携

- グラフ内のノードをクリックすると右解説パネルにノードの説明を表示する。
- SVG 内の要素クリックイベントを WebView2 の `window.chrome.webview.postMessage` 経由で ViewModel に伝達する（OQ-003）。

#### FR-VIZ-003 リアルタイム更新

- 真理値表・変数順序・種別が変更されるたびに図を自動的に再レンダリングする（短いデバウンスで連続入力を集約する）。
- 大きな図（ノード数 > 500）はレンダリング前に警告を表示する。

#### FR-VIZ-004 削減前後の切り替え

- グラフビューヘッダーのボタンで削減前（BDT）と削減後を切り替える。
- 切り替えと同時にステータスバーの統計値を更新する。

### 4.7 エクスポート

#### FR-EXP-001 TT コピー（真理値表クリップボードコピー）

- ツールバーの「TT コピー」ボタンで真理値表をクリップボードにコピーする。
- コピー形式（CSV / Markdown / AsciiDoc）を選択できる。
- `ToMarkdownTruthTable()` / `ToCsvTruthTable()` / `ToAsciiDocTruthTable()` を使用する。

#### FR-EXP-002 SVG エクスポート

- ツールバーの「SVG」ボタンで現在のグラフを SVG ファイルとして保存する。
- ファイル保存ダイアログを表示する。

#### FR-EXP-003 DOT エクスポート

- メニューまたはコンテキストメニューから DOT テキストをファイルに保存できる。

### 4.8 Undo / Redo

#### FR-UNDO-001 操作履歴の管理

- 真理値表の変更・変数順序の変更・図ファミリーの切り替えを Undo/Redo の対象とする。
- `Ctrl+Z` / `Ctrl+Y` キーボードショートカットに対応する。
- 履歴スタックの上限は 50 件とする。

### 4.9 設定

#### FR-SET-001 Graphviz パス設定

- Graphviz の `dot` 実行ファイルのパスを設定ページで指定できる。
- 起動時に自動検出を試みる（`PATH` および一般的なインストール先を確認）。

#### FR-SET-002 テーマ設定

- Light / Dark のテーマを選択できる。
- Tweaks パネルおよび設定ページの両方から変更可能にする。

#### FR-SET-003 ノード数上限設定

- `DecisionDiagramOptions.MaxNodeCount` を設定ページから変更できる。
- デフォルト値: 100,000。

## 付録 A. ライブラリ制約事項（UI への影響）

| ライブラリの制約 | UI への影響 |
|---|---|
| `BddManager`/`ZddManager` はスレッドセーフでない | 図操作は UI スレッドまたは専用シングルスレッドで実行する |
| 真理値表は変数数に対して指数的に増大する | 変数数 > 8 で上限警告を表示する。インライン表は 4 変数程度が上限推奨 |
| `EnumerateSets`/`EnumerateModels` に上限設定が必要 | デフォルト上限は 1,000 件、設定で変更可能 |
| `Core` は Graphviz に依存しない | DOT → SVG の変換はアプリ層（`GraphvizService`）で担う |
| `Core` は `netstandard2.0` ターゲット | `net8.0-windows` からのプロジェクト参照は問題なし |
| 削減前 BDT の生成 API はライブラリに存在しない | `DiagramService` がアプリ層で BDT を展開生成する（OQ-001） |
